🔐 安全说明

我们已经将邮箱订阅功能进行了安全重构：

  • 用户端：只能进行邮箱订阅操作
  • 管理端：需要密码验证，包含所有管理功能
  • API接口：提供标准的RESTful API
⚠️ 重要提醒： 订阅管理、导出配置等敏感功能已移至后台管理页面，普通用户无法访问。

👥 用户端功能

  • 简洁的订阅界面
  • 邮箱格式验证
  • 重复订阅检查
  • 响应式设计
  • 降级处理机制

🔧 管理端功能

  • 密码验证保护
  • 订阅统计信息
  • 订阅者列表管理
  • 搜索和过滤
  • 数据导出功能
  • 批量操作

🚀 API接口

  • POST /api/subscribe
  • POST /api/unsubscribe
  • GET /api/admin/subscribers
  • GET /api/health
  • 标准HTTP状态码
  • JSON响应格式

📋 使用步骤

  1. 启动服务：运行 npm run start-subscription
  2. 用户订阅：访问用户订阅页面进行邮箱订阅
  3. 管理订阅：使用管理员密码访问后台管理页面
  4. 导出数据：在后台管理页面导出订阅数据

🛡️ 安全配置
